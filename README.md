# Обзор примеров AI-агентских решений на базе локальных LLM

## 1.1 Использование локальной LLM через Ollama

Во всех трех примерах используется **локальная языковая модель (LLM)**, развернутая через **Ollama**:

```python
from langchain_ollama import ChatOllama
model = ChatOllama(model="qwen3:latest")
```

Это обеспечивает:
- Полную локальность выполнения (данные не покидают инфраструктуру)
- Использование современных моделей (qwen3, llama3, mistral)
- Гибкость в выборе моделей

## 1.2 Анализ реализаций примеров

### Agent_04_Drafter.py - Одиночный агент для документов
**Что реализовано:** Система для подготовки и редактирования документов

**Ключевые компоненты:**
- **Инструменты**: `update()` - обновление документа, `save()` - сохранение в файл
- **Граф состояний**: Линейный процесс с условными переходами
- **Логика работы**: 
  - Принимает запросы на изменение документа
  - Обновляет глобальную переменную `document_content`
  - Сохраняет результат по команде пользователя

**Архитектура:** Одиночный агент с инструментами

см. подробнее здесь: [README.md](Python/Agent_04_Drafter/README.md)

Папка с кодом здесь: [Agent_04_Drafter](Python/Agent_04_Drafter)

### supervisor_math_research.py - Мульти-агентная система с супервизором
**Что реализовано:** Система для решения математических и исследовательских задач

**Ключевые компоненты:**
- **Супервизор**: Управляет распределением задач между агентами
- **Специализированные агенты**:
  - `math_expert` - математические операции (сложение, умножение)
  - `research_expert` - поиск информации (имитация веб-поиска)
- **Инструменты**: `add()`, `multiply()`, `web_search()`

**Архитектура:** Иерархическая система с централизованным управлением

### swarm_flight_hotel.py - Роевая мульти-агентная система
**Что реализовано:** Децентрализованная система взаимодействующих агентов

**Ключевые компоненты:**
- **Агенты с индивидуальностью**:
  - `Alice` - эксперт по сложению, может передавать управление Bob
  - `Bob` - "пират", передает математические задачи обратно Alice
- **Инструменты передачи управления**: `create_handoff_tool()`
- **Хранилище состояния**: `InMemorySaver` для сохранения контекста диалога

**Архитектура:** Децентрализованная сеть равноправных агентов

## 1.3 Что такое AI Agent

**AI Agent** - это автономная система, которая:
- Использует LLM для принятия решений
- Имеет доступ к инструментам (функциям)
- Может выполнять последовательности действий для достижения цели
- Сохраняет состояние и контекст взаимодействия

Пример: `Agent_04_Drafter` - одиночный агент, управляющий документооборотом.

## 1.4 Multi-Agent vs Single AI Agent

| Аспект | Одиночный агент | Мульти-агентная система |
|--------|-----------------|-------------------------|
| **Архитектура** | Один агент со всеми инструментами | Несколько специализированных агентов |
| **Специализация** | Универсальный | Каждый агент имеет свою экспертизу |
| **Сложность** | Проще в реализации | Сложнее, требует координации |
| **Масштабируемость** | Ограничена | Легко добавлять новых агентов |

## 1.5 Типы Multi-Agent систем

### 1. Супервизорная система (supervisor_math_research.py)
- **Структура**: Иерархическая с центральным координатором
- **Управление**: Супервизор решает, какому агенту передать задачу
- **Преимущества**: Четкое разделение ответственности
- **Использование**: Когда задачи четко категоризируются

### 2. Роевая система (swarm_flight_hotel.py)
- **Структура**: Децентрализованная сеть
- **Управление**: Агенты самостоятельно передают задачи друг другу
- **Преимущества**: Гибкость, отказоустойчивость
- **Использование**: Для сложных, неструктурированных задач

## 1.6 Выводы

1. **Локальность через Ollama** обеспечивает приватность и контроль над данными
2. **Эволюция сложности**: от одиночного агента к сложным мульти-агентским системам
3. **Гибкость архитектур**: разные подходы под разные задачи
4. **LangGraph** предоставляет мощные абстракции для построения агентских систем
5. **Практическая применимость**: от автоматизации документооборота до сложных исследовательских задач

Все примеры демонстрируют зрелость экосистемы для построения production-готовых AI-решений на базе локальных моделей.
